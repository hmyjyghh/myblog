    # 低代码平台

    # 1. 低代码平台应用场景以及实际意义

    上面用一张图大致描述了下，低代码的应用场景及实际意义

    低代码平台基于组件化开发，旨在实现定制化，比如定制化门户首页，满足大多数用户需求

    # 2. 低代码/可视化搭建的核心思路

    • 先来了解一下传统的开发模式

    • 低代码平台开发模式

    看起来简化了不少，但是不要高兴的太早， 要做到通用配置和个性配置这两点，还是要依赖于底层开发的，只不过低代码平台开发完成之后，真是方便了很多

    开发的大致过程如下：

    1. 通用逻辑：因为要实现定制化，所以组件化开发，要尽可能抽象地具体，功能要全
    2. 可视化配置化：提供配置面板，让用户选择适合自己需求的配置
    3. 个性化接口，配置面板中填入接口地址，渲染对应的界面
    • 低代码平台的三大件

    低代码平台的核心布局，主要分为左中右三部分

    1. 左边叫物料堆，每做一个组件就往里面放一个，随着需求的扩大，数量会慢慢增加
    2. 中间叫主舞台，左侧物料堆里的组件经拖拽放入，主舞台，由渲染引擎解析，显示组件界面
    3. 右边是编辑面板，也可以称它为可视化面板配置界面，这里可以根据需求自主定义JSON配置项。

    # 5. 组件进阶批量模式管理

    1. 物料堆

    • 功能：内置组件和物料市场

      • 内置组件：标准化组件
      • 物料市场：由外部提供物料组件，给用户扩展能力，提供搜索和导入功能

    • 组件、布局组件、复合组件（元组件+布局组件）

      • 基础组件：元组件（不可嵌套）、布局组件（可嵌套）
      • 复合组件：由元组件 + 布局组件 + 复合组件组成

    • 开发设计：

      • 元组件：使用普通组件实现
      • 布局组件：slot或vnode，布局使用杉格（grid-layout）或flex布局或自定义绘制布局
    1. 主舞台

    • 所见及所得：渲染引擎一体化，优缺点：可探索，复杂度较高，逻辑集成度高

    • 多态舞台：分为配置舞台和渲染引擎，优缺点：状态分离、效率高、同时两拨人维护，eg：配置使用h5，渲染为小程序后flutter。

    1. 编辑面板(也叫配置面板)
    • 提供配置项， 可供选择
    1. 工具栏
    • 保存
    • 预览
    • 重做

    # 6. 实现

    技术选型：vue、 react jsx模板

    生态：组件库、路由、状态管理等

    1. 布局：顶栏、物料区、主舞台、配置区：el-container、el-header、el-main(el-row、el-col)

    2. 物料区：元组件和布局组件，contanier、cInput、cButton

    3. 主舞台：封装渲染引擎

    4. 添加渲染组件层，与元组件解耦合，使用require.context自动导入组件和渲染组件使用

    5. 定义vnode树形数据结构，渲染函数逐级解析vnode树，最终调用渲染组件的render函数进行渲染，可使用jsx简化渲染函数。

    6. 物料区和主舞台联动的拖拽实现：使用h5 draggable属性、ondrop事件(@dragover.prevent)、ondragstart事件实现

    一. 不使用渲染引擎：渲染可使用动态组件切换，使用selectType变量或者event确定拖拽组件是哪一个

    <template>
  <div>
<!-- 物料区 -->
  <ul>
    <li
    v-for="item in stack"
    :key="item.code"
    class="component"
    :draggable="true"
    @drag="handleDrag(item)"
    >
    {{ item.value }}
    </li>
 </ul>
  <!-- 主舞台 -->
  <div class="stage block" @dragover.prevent @drop.stop="handleDrop">
    <div v-for="(item, index) in components" :key="index">
        {{ item }}
      <component :is="item" />
    </div>
  </div>
 </div>
</template>
<script>
export default {
  methods:{
    handleDrag (item) { // 拾取被配置节点
      this.selectedType = item
    },
    handleDrop () { // 放手
      const _type = this.selectedType
      this.components.push(_type)
    },
  },
  data(){
    return {
      components:[], //组件数组
    }
  },
  components:{
    ...modules //动态组件使用，需引入全部对象
  }
}
</script>

    二. 使用渲染引擎，渲染引擎传入jsonSchema、addNode数据，container组件、parser-container组件、渲染引擎添加事件处理。

    1. 全部放置在container根组件下
    2. container中添加drop事件，通过事件抛出，向渲染引擎推送被放入的container组件信息
    3. 渲染引擎接收到事件后，对container组件的jsonSchema添加children达到添加组件的目的
    <script>
import { components, parsers } from '../components'
export default {
  name: 'renderEngine',
  props: {
    // 配置协议
    jsonSchema: {
      type: Object,
      default: () => {
        return {}
      }
    },
    // 新增拖入舞台的节点
    addNode: {
      type: String,
      default: ''
    }
  },
  data () {
    return {
      page: undefined
    }
  },
  methods: {
    init () {
      // 提取页面参数（单次初始化）
      this.page = this.jsonSchema.page || {}
    },
    // 渲染根节点
    renderRoot (h) {
      let _page = this.page

      // TODO: 后期丰富全局配置逻辑入口
      return (
        <div class="root">
          { this.renderComponents(h, _page) }
        </div>
      )
    },

    // 渲染组件
    renderComponents (h, section) {
      // 组件通用逻辑在此处理
      // 是否有子节点
      let _children = null

      if (section.children) {
        // 层级渲染
        _children = this.renderChildren(h, section)
      }
      return this.startRender(h, section, _children)
    },

    // 遍历包含兄弟&子节点
    renderChildren (h, section) {
      let _nodeArray = section.children || [].concat(section)
      // 后期可以在此拓展兄弟节点之间通信
      return _nodeArray.map((n, i) => this.renderComponents(h, n, i))
    },

    // 开始渲染
    startRender (h, section, _children) {
      const _type = section.type
      const renderMod = parsers[_type]

      // 直接渲染
      if (renderMod) {
        return renderMod.render.call(this, h, section, _children)
      }
      return null
    },

    // 以下为配置系统统一化处理逻辑
    // 拖拽组件经过触发
    handleDragOver () {
    },
    // 拖拽组件松手
    handleDrop (event, vm) {
      const _json = vm.jsonSchema
      // console.log('_json', _json);

      if (_json && (_json.type === 'Container' || _json.type === 'GRidone')) {
        // console.log('_json', _json.type);

        if (!_json.children) {
          this.$set(_json, 'children', [])
        }
        _json.children.push({
          type: this.addNode
        })
      }
    }
  },
  components: {
    ...components,
    ...parsers
  },
  created () {
    this.init()
  },
  mounted () {
    // console.log('...components', components);
  },
  render (h) {
    let _vode = this.renderRoot(h)

    return _vode
  }
}
</script>
    1. 配置面板

    一个组件对应一个配置面板，设置config文件夹，自动导入配置面板组件

    1. 添加configPanel组件，添加配置面板组件，根据输入的pickType数据，自动渲染该类型的配置面板。
    2. parse-xx组件添加click原生监听事件，点击时触发pickType事件，发送配置类型数据。
    3. 主面板mainPage中对parse-xx的父组件渲染组件添加pickType事件，回调函数中修改当前currentPickType值，传给configPanel更新。
    <script>
  export default {
    name: 'configPanel',
    props: {
      currentPickType: {
        type: String,
        default: 'cButton'
      },
      data: {
        type: Array,
        default () {
          return []
        }
      }
    },
    components: {
      // ...components
      // DsfDesignerProperties
    },
    methods: {
      // vue的实例属性$options是用来获取定义在data外的数据和方法的。
      renderPanel (h, type) {
        if (!type) return
        const components = this.$options.components
        // console.log('components', components);
        // console.log('components[type]', components[type]);
        // console.log('函数', components[type].render);

        return (
          components[type].render.call(this, h)
        )
      }
    },
    render (h) {
      // console.log('h', h);
      // 初始化参数
      const _type = this.currentPickType
      console.log('_type', _type)
      // 渲染面板
      let _panel = this.renderPanel(h, _type)

      return _panel
    }
  }
</script>

    # 7. 总结

    • 物料堆

      • 使用require.context, 动态引入物料堆的组件
      • 后续只需往里丢入组件，无需调整物料加载代码

    • 渲染引擎

      • 拿到jsonSchema协议
      • 遍历解析, 渲染根节点
      • 调用渲染组件的函数, 判断是否有子节点，有的话，调用renderChildren函数，递归执行渲染组件的函数
      • 直接渲染，调用组件的render函数，挂载组件
      • 完成解析
