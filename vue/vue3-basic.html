    # 5. watchEffect

    它是一个副作用函数，会主动帮我们创建一个副作用函数

    • 不需要指定，监听哪个值
    • 内部包含的任意一个响应式数据发生改变，都会去触发，自动去执行这个副作用函数
    import { ref, reactive, watchEffect } from 'vue'

watchEffect(() => {
  // 此处 count 或者 myData 中的age  发生改变  都会执行这个函数
  console.log(count.value, myData.age)
})

    # 6. script setup 有什么特点?

    • 无需手动导入 defineComponent 和 setup() 函数，直接在 <script setup> 中编写逻辑即可‌。

    • ‌自动处理组件定义‌：所有顶层变量、函数、导入的组件等均自动暴露给模板，无需通过 return 显式导出‌。

    • 原理

    
setup(_, { expose }) {
  expose()

  const _returned = { msg, count }

  return _returned
}

// expose 是用来暴露实例成员的
    1. setup 函数中，如何暴露实例成员
    setup(_, { expose }) {

  // 暴露指定想暴露的东西
  expose({ msg, count })
}
    1. setup语法糖中，如何暴露实例成员
    // 通过 defineExpose 这个  宏，只在编译状态时存在
<script setup>

// 宏
defineExpose({ 
  msg, 
  count
}) 
// 编译后还是会转成 expose 函数
</script>

    # 7. vue2和vue3区别?

    1. 新增了Composition API
    2. 支持多个根节点，增加了teleport 组件
    3. 虚拟DOM, 响应式系统优化，使用Proxy 替代了Object.defineProperty
    4. 增加了对ts 的支持
    5. 新的生命周期函数，onMounted、onBeforeUpdate 等
    6. Vue 3 源码使用 ES Modules (ESM) 编写, 按需引入，打包优化，有利于 tree-shaking

    # 13. vue3中如何创建一个可以操作的 computed

    使用 getter 和 setter 函数, 可操作的计算属性具备 getter 和 setter 函数，通过它们可以对计算属性的值进行读取和设置。

    <template>
  <div>
    <p>Full Name: {{ fullName }}</p>
    <input v-model="fullName" placeholder="Enter full name">
  </div>
</template>

<script setup>
import { ref, computed } from 'vue';

// 定义 firstName 和 lastName
const firstName = ref('John');
const lastName = ref('Doe');

// 创建可操作的计算属性 fullName
const fullName = computed({
  // getter 函数，返回完整姓名
  get() {
    return `${firstName.value} ${lastName.value}`;
  },
  // setter 函数，根据输入更新 firstName 和 lastName
  set(newValue) {
    const names = newValue.split(' ');
    firstName.value = names[0];
    lastName.value = names[1] || '';
  }
});
</script>

    # 常见对比

    # 1. ref 和 reactive 的区别

    • ref：常用于创建单个值的响应式数据，像基本数据类型（如 number、string、boolean）或者需要在模板中使用的变量。
    • reactive：适合创建复杂的对象或数组的响应式数据。

    # 2. toRef 和 toRefs

    • toRef toRef 用于为源响应式对象上的属性新建一个ref，从而保持对其源对象属性的响应式连接。

    • 接收两个参数：源响应式对象和属性名，返回一个ref数据。

    • 例如使用父组件传递的props数据时，要引用props的某个属性且要保持响应式连接时就很有用。

    import { defineComponent, toRef } from 'vue'

export default defineComponent({
  props: [title],
  
  setup (props) {
    // 创建变量myTitle
    const myTitle = toRef(props, 'title')

    console.log(myTitle.value)
  }
})

    • 获取数据值的时候需要加.value

    • toRef后的ref数据如果是复杂类型数据时，不是原始数据的拷贝，而是引用，改变结果数据的值也会同时改变原始数据

    • toRefs

    1. toRefs 可以把一个响应式对象转换为普通的对象。
    2. 该普通对象的每一个值都是ref。
    3. 由于变成了ref，所以我们需要使用value。
    setup(){
  let obj=reactive({
    name:'张三',
    age:30,
    sex:'男'
  })

  return {
    ...toRefs(obj)
  }
}

    toRef是把对象的某个属性变成响应式的，toRefs是把对象的每个属性都变成响应式的

    # 3. vue3中isRef、unref、toRef、toRefs用法

    1. isRef: 检查某个值是否为ref, 也就是看是不是用ref 定义的，如果是用reactive 定义的对象，则返回false

    2. unref: 如果参数是ref,则返回内部值,否则返回参数本身

    3. toRef:

    • 基于响应式对象上的一个属性,创建一个对应的ref。
    • 这样创建的ref与其源属性保持同步:
    • 改变源属性的值将更新 ref 的值,反之亦然。
    1. toRefs
    • 将一个响应式对象转换为一个普通对象,
    • 这个普通对象的每个属性都是指向源对象相应属性的ref,
    • 每个单独的ref都是使用toRef()创建的。
