    # uniapp 相关

    # 一、基础概念

    # 1. UniApp 的跨平台原理是什么？

    • 核心机制：
      • 通过条件编译（#ifdef H5、#ifdef MP-WEIXIN）生成不同平台的代码。
      • 运行时基于 Vue.js 语法，底层调用各平台原生 API（如小程序 wx.xxx、H5 的 DOM）。
    • 渲染差异：
      • 小程序端：将 Vue 模板转译成 WXML/JS。
      • H5 端：直接运行 Vue 的虚拟 DOM 渲染。
      • App 端：通过 Weex 或原生渲染（需区分 vue 和 nvue）。

    # 2. UniApp 的生命周期有哪些？和 Vue 的区别？

    • 页面生命周期（小程序风格）：
      onLoad、onShow、onReady、onHide、onUnload。
    • 组件生命周期（Vue 风格）：
      beforeCreate、created、mounted、updated、destroyed。
    • 区别：
      • UniApp 新增了小程序特有的生命周期（如 onPullDownRefresh）。
      • App 端支持 onBackPress（返回按钮监听）。

    在 UniApp 项目里，App.vue 中的 onLaunch 和组件里的 onLoad 是生命周期函数，有着不同的执行机制。

    # App.vue 中的 onLaunch

    • 触发时机：在小程序或应用启动时，onLaunch 函数会最先执行，而且只会执行一次。这是整个应用生命周期的起始点，适合进行一些全局的初始化操作，像全局变量的初始化、用户登录状态的检查等。
    • 作用范围：onLaunch 属于全局应用级别的生命周期函数，其作用范围涵盖整个应用。
    • 参数情况：onLaunch 可以接收到小程序启动时的参数，比如从分享链接进入时携带的参数。

    # 组件里的 onLoad

    • 触发时机：当组件被创建并载入页面时，onLoad 函数会被触发。对于页面组件而言，每次页面加载时都会执行；对于自定义组件，在组件被创建并挂载到页面时执行。
    • 作用范围：onLoad 是组件级别的生命周期函数，其作用范围仅限于当前组件。
    • 参数情况：onLoad 可以接收到页面跳转时传递的参数，通过这些参数能够对组件进行动态初始化。

    # 执行顺序

    在应用启动时，先执行 App.vue 中的 onLaunch，之后再根据页面加载情况执行相应组件的 onLoad。

    // 在某个 js 文件中
App({
	onLaunch() {
		console.log('应用启动了');
    // 可以在这里进行全局初始化操作
    // 例如：检查用户登录状态、初始化全局变量等
	}
})


// 页面组件

onLoad(options) {
	console.log('页面加载了', options);
	// 可以在这里根据传递的参数进行页面初始化
}

    在这个示例中，应用启动时会先输出 “应用启动了”，之后当首页加载时会输出 “页面加载了” 以及传递的参数。

    # 二、开发实践

    # 3. 如何解决跨平台样式兼容问题？

    • 方案：
      • 使用 rpx 替代 px 实现响应式布局。
      • 条件编译样式：
        /* #ifdef H5 */
.box { margin: 10px; }
/* #endif */
      • 避免使用 CSS 高级特性（如 position: sticky 在小程序不兼容）。

    # 4. UniApp 如何实现全局状态管理？

    • 推荐方式：
      • Vuex：适合复杂应用（需在 main.js 中挂载）。
      • uni.$on/uni.$emit：简单事件通信。
      • App.vue 的 globalData：
        // App.vue
export default {
  globalData: { userInfo: null }
}
// 页面中使用
const app = getApp();
console.log(app.globalData.userInfo);

    # 5. 如何优化 UniApp 的启动速度？

    • 策略：
      • 减少首包体积：分包加载（subPackages）。
      • 延迟加载非必要组件：v-if 或动态导入。
      • 启用 v3 编译模式（优化渲染性能）。
      • App 端使用 nvue 替代 vue 提升性能。

    # 三、原生交互

    # 6. 如何调用原生设备功能（如摄像头、GPS）？

    • 方案：
      • 统一 API：uni.chooseImage、uni.getLocation。
      • Native.js（App 端）：直接调用原生 Java/OC 代码。
      • 插件市场：集成第三方原生插件（如扫码、支付）。

    # 7. UniApp 如何与原生 App 通信？

    • App 端：
      • JS Bridge：uni.postMessage 和 uni.onMessage。
      • WebView 通信：通过 URL Scheme 或 evalJS。
    • 小程序端：
      • 使用 uni.navigateToMiniProgram 跳转其他小程序。

    # 四、性能优化

    # 8. 列表渲染卡顿怎么优化？

    • 解决方案：
      • 使用 虚拟列表（如 uni-list 组件）。
      • 避免在 v-for 中复杂计算，提前处理数据。
      • 分页加载：onReachBottom 监听滚动到底部。
      • App 端使用 nvue 的 list 组件（原生渲染性能更高）。

    # 9. 如何减少白屏时间？

    • 措施：
      • 启用 骨架屏（通过 uni-skeleton 插件）。
      • 预加载关键数据：在 onLoad 中提前请求。
      • 使用 v3 编译模式 的 按需注入 减少初始化耗时。

    # 五、调试与发布

    # 10. 如何调试 UniApp 的跨端问题？

    • 工具：
      • H5：Chrome 开发者工具。
      • 小程序：各平台开发者工具（微信/支付宝）。
      • App 端：HBuilderX 的真机调试 + console 日志。
    • 技巧：
      • 使用 uni.getSystemInfo 打印环境信息。
      • 条件编译调试：
        // #ifdef DEBUG
console.log('调试信息');
// #endif

    # 11. 如何实现热更新？

    • App 端：
      • 使用 uni-app 官方提供的 wgt 包更新（静默或提示用户）。
      • 第三方方案：如 DCloud 的 uni-upgrade-center。
    • 小程序/H5：
      • 小程序需重新提交审核，H5 直接更新服务器资源。

    # 六、高级特性

    # 12. UniApp 的 nvue 和 vue 有什么区别？

    特性 vue nvue
    渲染引擎 WebView 渲染 原生渲染（Weex 或 Uni-Runtime）
    性能 较低（适合复杂 UI） 更高（适合长列表、动画）
    CSS 支持 完整 CSS 受限（如不支持 flex-wrap）
    适用场景 多端兼容 App 端高性能需求

    # 13. 如何实现一个跨平台的 TabBar？

    • 方案：
      • 使用 uni-app 内置的 uni-tabbar 组件。
      • 自定义 TabBar：
        • 通过 条件编译 适配不同平台样式。
        • 监听 onTabItemTap 处理点击事件。

    # 七、面试加分项

    # 14. UniApp 的底层是如何打包的？

    • 流程：
      1. 通过 webpack 将 Vue 代码转译成各平台格式。
      2. 小程序端：生成 WXML/WXSS/JS。
      3. App 端：打包为 apk/ipa 或 wgt 热更新包。
    • 关键配置：
      • manifest.json 控制应用名称、图标等。
      • pages.json 配置路由、导航栏样式。

    # 15. UniApp 3.0 有哪些改进？

    • Vue 3 支持：Composition API + 更好的 TS 支持。
    • 性能提升：v3 编译器优化、更快的渲染速度。
    • 新特性：uni-cloud 云开发集成、uni-ui 组件库增强。

    # 总结回答示例

    UniApp 如何实现跨平台？

    “UniApp 通过条件编译将同一套代码转译成多端格式，

    • 例如小程序转译为 WXML
    • H5 直接运行 Vue。运行时根据平台调用对应 API，如 uni.request 在小程序端映射为 wx.request。
    • 对于渲染差异，App 端支持 vue（WebView）和 nvue（原生）两种模式，开发者可根据性能需求选择。”

    # 1. 什么是 uniapp？核心特点是什么？

    uniapp 是 DCloud 推出的跨端开发框架，基于 Vue.js，支持一套代码编译到 iOS、Android、H5、微信 / 支付宝等小程序、快应用等多平台。

    • 核心特点
      • 跨端兼容，一套代码多端运行；
      • 支持 Vue 语法，上手门槛低；
      • 兼容微信小程序 API，无需二次适配；
      • 内置丰富 UI 组件（如uni-ui）；
      • 支持原生渲染（nvue）和 WebView 渲染。

    # 2. 如何实现多端兼容？

    1. 条件编译
    <!-- #ifdef H5 -->
<view>H5页面</view>
<!-- #endif -->
    1. 统一 API：使用 uniapp 封装的跨端 API（如uni.request），减少平台差异。

    # 3. uniapp 的运行模式有哪些？

    • H5：npm run dev:h5，浏览器调试；
    • 小程序：npm run dev:mp-weixin（微信为例）；
    • App：npm run dev:app，支持 iOS/Android。

    # 4. 如何进行页面跳转？

    • 保留当前页：uni.navigateTo({ url: '/pages/detail' })；
    • 关闭当前页：uni.redirectTo({ url: '/pages/home' })；
    • 跳转到 TabBar 页：uni.switchTab({ url: '/pages/tab' })。

    # 5. 如何实现全局状态管理？

    • Vue2：使用 Vuex；
    • Vue3：推荐 Pinia（轻量且支持组合式 API）。

    # 6. 如何优化小程序包体积？

    • 减少不必要的 npm 包；
    • 开启代码压缩（Webpack 配置）；
    • 使用分包加载（subPackages配置）。

    # 7. 如何提高渲染性能？

    • 减少setData次数（Vue 组件已优化）；
    • 使用scroll-view代替list；
    • 图片懒加载（lazy-load属性）。

    # 8. 为什么v-html不能渲染视频？如何解决？

    原因：小程序端限制，无法解析<video>标签。

    • 解决方案：
    1. 使用rich-text组件；
    2. 通过wxParse等库解析 HTML。

    # 9. uniapp 如何实现跨端编译？

    • 使用统一 Vue 代码，通过uni-app compiler编译为不同端代码：
      • H5：Vue+Webpack；
      • 小程序：转换为原生代码；
      • App：使用 Weex 或 nvue 渲染。

    # 10. 其他高频问题

    • 数据绑定：使用双花括号{{}}或v-model。
    • 数据缓存：uni.setStorageSync(key, value)。
    • 下拉刷新：在页面生命周期onPullDownRefresh中调用uni.stopPullDownRefresh()。
    • 地理位置：uni.getLocation({ success(res) { ... } })。

    # uniapp和vue的异同点

    # 1. 组件/标签的变化

    • p 改成 view
    • span、font 改成 text
    • a 改成 navigator
    • img 改成 image
    • input 还在，但type属性改成了confirmtype
    • form、button、checkbox、radio、label、textarea、canvas、video 这些还在。
    • select 改成 picker
    • iframe 改成 web-view
    • ul、li没有了，都用view替代
    • audio 不再推荐使用，改成api方式，背景音频api文档

    虽然老的HTML标签也可以在uni-app里使用，uni-app编译器会在编译时把老标签转为新标签，比如把p编译成view。但不推荐这种用法，调试H5端时容易混乱。

    # 2. 新增了手机端常用的新组件

    • scroll-view 可区域滚动视图容器
    • swiper 可滑动区域视图容器
    • icon 图标
    • rich-text 富文本（不可执行js，但可渲染各种文字格式和图片）
    • progress 进度条
    • slider 滑块指示器
    • switch 开关选择器
    • camera 相机
    • live-player 直播
    • map 地图
    • cover-view 可覆盖原生组件的视图容器

    cover-view 注意事项

    uni-app的非h5的video、map、canvas、textarea是原生组件，层级高于其他组件。如需覆盖原生组件，比如在map上加个遮罩，则需要使用cover-view组件

    # 3. 和浏览器的js api的不同

    • alert,confirm 改成 uni.showToast, uni.showModel
    • ajax axios 改成 uni.request
    • cookie、session 没有了，local.storage 改成 uni.storage
    • vue-router没有了，改成了 uni.navigateTo

    # 4. uniapp自带路由和请求方式

    • uni.navigateTo 路由与页面跳转
    • uni.request 网络请求

    # 5. uniapp中增加的钩子函数

    • onReady()
    • onShow()
    • onHide()

    # 6. upx rpx px

    upx rpx 响应式单位

    当需要某元素的单位要根据屏幕宽度（小范围）大小变化时，才需要rpx这类动态宽度单位。

    # 相同点

    1. uni-app中可以使用 created、mounted、beforeDestroy 等钩子函数

    2. 可以使用vue中的options API

    # 通用API封装

    实际开发中频繁使用到的操作, 我们可以封装成一些通用的插件

    # 1. 弹窗API

    uni-app 提供的API有 uni.showToast

    // 示例1
Vue.prototype.$showToast = function(msg, image) {
	uni.showToast({
		title: msg,
		icon: 'none',
		image: image ? image : ""
	})
}

// 示例2
Vue.prototype.$toast = function(title, icon = 'none') {
	uni.showToast({
		title,
		icon
	})
}

this.$toast('保存成功', 'success')

    # 2. 请求API

    1. 基于 uni.request(OBJECT) 进行封装
    2. 定义一个对象 或者 类，方便导出使用
    3. 定义一个封装请求的方法
    4. 接收一个对象，包含url,method,header, data,success callback, error callback 等属性
    5. 调用 uni.request
    let obj = {
	_request: function(params) {
		let _params = {}
		_params.url = params.url
		_params.header = params.header
		_params.data = params.data
		_params.method = params.method
		_params.success = params.success
		_params.fail = params.fail

		uni.request({
			url: _params.finalUrl,
			data: _params.data,
			header: _params.header,
			method: _params.method,
			success: function(res) {

			},
			fail: function(err) {

			},
			compelete: function() {

			}
		})
	}
}

    # 常见的业务场景

    # 1. 列表页面

    列表页面一般包含如下功能：

    下拉刷新，上滑加载，没有更多内容的提示，关键字搜索，标签栏匹配，空白页面，列表项组件

    项目中一般都需要复用这些组件

    # 2. 跨页面方法调用

    点击搜索，跳转到搜索页面后，点击搜索，我们需要刷新当前页面，这个时候我们就需要一个在当前页面调用上一个页面的方法

    searchFunc(){
  let pages = getCurrentPages()
  let page = pages[pages.length - 2]
  page.$vm.searchText = this.search.text
  page.$vm.refresh()
  this.router.back()
}

    getCurrentPages() 是一个全局函数，用于获取当前页面栈的实例，以数组形式按栈的顺序给出。我们当前页面就是数组最后一项，那么上一个页面就是pages[pages.length - 2]。当然我们还要记得加上$vm属性，因为在uniapp中我们的数据和方法是挂载这个实例上的。我们可以通过这个示例访问到对应页面的所有数据和方法！

    # 4. 默认使用高德导航APP进行路线导航

    IOS端 uni.openLocation, 默认调用苹果自带Apple地图，怎么默认百度、高德等地图导航

    需要注意：

    1. iOS上获取本机是否安装了百度高德地图，需要在manifest里配置，在manifest.json文件app-plus->distribute->ios-urlschemewhitelist节点下添加 "iosamap"

    注意：宿主APP 也要设置百度、高德地图的白名单(才能成功跳转第三方应用)

    1. 如果还需要百度地图，和腾讯地图等，则添加对应的类型，baidumap,qqmap
    // 导航
navigationTo(item) {
	//打开第三方地图
	const latitude = Number(item.latitude)
	const longitude = Number(item.longitude)
	if (plus.os.name == 'Android') {
		uni.openLocation({
			latitude: latitude,
			longitude: longitude,
			name: item.siteDetail,
			scale: 12,
			success() {
				console.log('success')
			}
		})
	} else { // ios默认打开高德地图
		this.iosOpenMap(latitude, longitude, item.siteDetail)
	}
},
iosOpenMap(latitude, longitude, name) {
	let url = ''
	plus.nativeUI.actionSheet({
		title: "选择地图应用",
		cancel: "取消",
		buttons: [{
			title: "高德地图"
		}]
	}, function(e) {
		switch (e.index) {
			case 1:
				url = `iosamap://viewMap?sourceApplication=applicationName&poiname=${name}&lat=${latitude}&lon=${longitude}&dev=0`;
				break;
			default:
				break;
		}
		if (url != "") {
			url = encodeURI(url);
			plus.runtime.openURL(url, function(e) {
				plus.nativeUI.alert("本机未安装指定的地图应用");
			});
		}
	})
}
